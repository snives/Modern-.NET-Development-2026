# Manuscript Instructions

These instructions apply to all content under `/book`.

Before drafting a chapter:

1. Read the book roadmap.
2. Read the preceding chapter.
3. Read the chapter specification.
4. Identify what the reader already knows at this point.
5. Identify concepts that must not be introduced yet.
6. Produce or update the chapter outline before writing prose.

Do not introduce unexplained terminology.

When a term is first introduced:

- explain why it exists;
- relate it to the reader's established Windows/.NET mental model;
- show where it sits in the overall architecture;
- then explain how it works.

Avoid repeating complete explanations from earlier chapters. Link back to the
earlier chapter and state only what is necessary for the current discussion.

Use the structure defined in `chapter-template.md`.
